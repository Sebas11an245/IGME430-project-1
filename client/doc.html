Pokedex API Documentation

This API provides access to a Pokémon dataset. Users can retrieve Pokémon using GET requests and modify the dataset using POST requests. All responses are returned in JSON format.

Base URL

http://localhost:3000

GET /api/pokemon

Returns a list of Pokémon. Supports query parameters.

Query Parameters:

  • name (optional) – filter by Pokémon name
  • type (optional) – filter by Pokémon type

Example:

/api/pokemon?type=Fire

Status Codes:

  • 200 – Success
  • 204 – No results found

GET /api/pokemonById

Returns a single Pokémon by ID.

Query Parameters:

  • id (required) – Pokémon ID number

Example:

/api/pokemonById?id=25

Status Codes:

  • 200 – Success
  • 400 – Invalid ID
  • 404 – Pokémon not found

GET /api/types

Returns a list of all Pokémon types in the dataset.

Status Codes:

  • 200 – Success

GET /api/random

Returns one random Pokémon from the dataset.

Status Codes:

  • 200 – Success

POST /api/addPokemon

Adds a new Pokémon to the dataset.

Body Parameters:

  • id – number
  • name – string
  • type – array or comma-separated string

Accepted Formats:

  • application/json
  • application/x-www-form-urlencoded

Status Codes:

  • 201 – Created
  • 400 – Missing or invalid data

POST /api/updatePokemon

Updates an existing Pokémon’s name, ID, or type.

Body Parameters:

  • id – ID of Pokémon to update
  • name (optional)
  • type (optional)

Status Codes:

  • 200 – Updated successfully
  • 400 – Invalid data
  • 404 – Pokémon not found

HEAD Requests

All GET endpoints also support HEAD requests. HEAD requests return the same headers as GET requests but do not include a response body.

Response Format

All responses are returned in JSON format and include appropriate HTTP status codes.

Client Usage

The client application uses the Fetch API to send GET, POST, and HEAD requests to the server and dynamically updates the page with Pokémon data.
